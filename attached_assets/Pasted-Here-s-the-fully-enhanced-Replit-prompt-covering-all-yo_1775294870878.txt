### PART 1 — CRITICAL BUG FIX: Events Cards Going Blank

**Problem:** In the Events section, when the user clicks the "Past" filter tab, all event cards go completely blank/white. If they then switch back to "All" or "Upcoming", those cards also go blank until hovered. This is caused by the `.reveal` IntersectionObserver running only once on mount — when filter state changes and cards re-render, they restart with `opacity: 0` but are never re-observed.

**Fix:** In `App.tsx`, update the `useReveal` hook and the `Events` component as follows:

1. Change `useReveal()` to accept a dependency array and re-run on filter changes:
```tsx
function useReveal(deps: React.DependencyList = []) {
  useEffect(() => {
    const observer = new IntersectionObserver(
      (entries) => {
        entries.forEach((e) => {
          if (e.isIntersecting) {
            e.target.classList.add("visible");
            observer.unobserve(e.target); // stop watching after visible
          }
        });
      },
      { threshold: 0.05 }
    );
    // Small delay to allow DOM to update after filter change
    const timeout = setTimeout(() => {
      const els = document.querySelectorAll(".reveal:not(.visible)");
      els.forEach((el) => observer.observe(el));
    }, 50);
    return () => {
      clearTimeout(timeout);
      observer.disconnect();
    };
  }, deps);
}
```

2. In the `Events` component, call `useReveal([filter])` so it re-runs every time the filter changes.

3. Also fix the event cards: when the filter changes, remove the `visible` class from all event cards first so they can re-animate cleanly. Add a `useEffect` inside `Events`:
```tsx
useEffect(() => {
  // Reset visibility of event cards on filter change so reveal re-runs
  document.querySelectorAll(".event-card.reveal").forEach(el => {
    el.classList.remove("visible");
  });
}, [filter]);
```

4. In the root `App` component, keep `useReveal([])` only for static sections (About, Programs, etc.) — not for the Events section which handles its own reveal.
